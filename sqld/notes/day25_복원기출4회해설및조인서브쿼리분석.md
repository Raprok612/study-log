## **🧠 이론 정리**

### **1️⃣ 조인(Join) 복습**

| 유형 | 설명 | 예시 |
|------|------|------|
| **INNER JOIN** | 교집합(공통 데이터만 조회) | `SELECT * FROM A JOIN B ON A.ID = B.ID` |
| **LEFT OUTER JOIN** | 왼쪽 기준, 오른쪽 NULL 포함 | `A LEFT JOIN B ON A.ID = B.ID` |
| **RIGHT OUTER JOIN** | 오른쪽 기준 | `A RIGHT JOIN B ON A.ID = B.ID` |
| **SELF JOIN** | 자기 자신과 조인 | `EMP E1 JOIN EMP E2 ON E1.MGR = E2.EMPNO` |
| **FULL OUTER JOIN** | 양쪽 모두 포함 | `A FULL OUTER JOIN B ON A.ID = B.ID` |

> 💡 시험에서는 LEFT JOIN + NULL 조건 결합이 자주 출제됨.

---

### **2️⃣ 서브쿼리(Subquery)**

| 구분 | 설명 | 예시 |
|------|------|------|
| **단일행 서브쿼리** | 단일 결과 비교 | `SAL > (SELECT AVG(SAL) FROM EMP)` |
| **다중행 서브쿼리** | 여러 결과 비교(IN / ANY / ALL) | `SAL > ALL (SELECT SAL FROM EMP)` |
| **상관 서브쿼리** | 외부 쿼리의 값을 내부 쿼리에서 사용 | `WHERE SAL > (SELECT AVG(SAL) FROM EMP B WHERE B.DEPTNO = A.DEPTNO)` |

> ⚙️ 상관 서브쿼리는 실행 효율이 낮아 성능 관점에서도 주의.

---

## **🧮 예제 및 실습**

**예제 1️⃣ – INNER JOIN + 조건 필터**
```sql
SELECT E.ENAME, D.DEPT_NAME
FROM EMP E
JOIN DEPT D ON E.DEPTNO = D.DEPTNO
WHERE E.SAL >= 3000;
```

**예제 2️⃣ – LEFT JOIN + NULL 처리**
```sql
SELECT D.DEPT_NAME, E.ENAME
FROM DEPT D
LEFT JOIN EMP E ON D.DEPTNO = E.DEPTNO
WHERE E.ENAME IS NULL;
```

**예제 3️⃣ – 상관 서브쿼리 실습**
```sql
SELECT ENAME, SAL
FROM EMP A
WHERE SAL > (SELECT AVG(SAL) FROM EMP B WHERE A.DEPTNO = B.DEPTNO);
```

- [x] JOIN 유형별 결과 비교  
- [x] NULL 처리 + LEFT JOIN 실습  
- [x] 상관 서브쿼리 조건 실전 풀이
