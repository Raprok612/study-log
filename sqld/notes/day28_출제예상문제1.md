# SQLD Day 28 – 1장 출제예상문제 풀이 정리 (2025-11-09)

## 🎯 학습 목표
- 아이리포 SQLD 1장 출제예상문제(34–35강) 핵심 개념 복습
- 데이터모델링 이론을 문제 관점에서 구조화해 정리
- 자주 출제되는 패턴 및 함정 보기 유형 숙지

## 🧠 이론·개념 재정리 (1장 핵심 회복습)

### 1️⃣ 데이터모델링의 개념과 역할
- 현실 세계의 업무 데이터를 구조화해 표현하는 과정
- 업무 규칙·데이터 의미·관계를 명확하게 정의하는 작업
- “무엇을 데이터로 관리할 것인가”에 대한 기준을 세움

**핵심 키워드**
- 추상화 / 단순화 / 명확화
- 데이터 중복 최소화, 비일관성 제거

### 2️⃣ 데이터모델링의 3단계 구조 (개념·논리·물리)

| 단계 | 관점 | 주요 내용 |
| --- | --- | --- |
| 개념 모델 | 업무 중심 | 엔터티, 관계, 속성 도출 |
| 논리 모델 | 구조 중심 | 정규화, 키 정의, 상세 속성 설계 |
| 물리 모델 | 구현 중심 | 테이블/인덱스/파티션 등 실제 DB 구조 |

### 3️⃣ 엔터티·속성·관계 핵심 포인트
- **엔터티**: 업무에서 지속적으로 관리해야 할 객체(식별자 필수)
- **속성**: 엔터티를 설명하는 최소 단위 데이터
- **관계**: 엔터티 간 연관성 (1:1 / 1:N / N:M)

## ❗ 1장 출제예상문제 – 자주 나오는 패턴

### 1️⃣ 좋은 엔터티 / 특징 문제
**정답 패턴**
- 두 개 이상의 인스턴스를 포함
- 두 개 이상의 속성을 가짐
- 유일한 식별자 존재

**오답 패턴**
- “반드시 물리 테이블이어야 한다”
- “하나의 인스턴스만 있어도 엔터티이다”

### 2️⃣ 모델링 목적·장점 보기 유형
- 중복 최소화 / 일관성 확보 / 의미 명확화 → 정답 패턴
- 성능 향상이 목적이다 → 단독으로 나오면 오답

### 3️⃣ 스키마 관련 문제
- 외부: 사용자 관점
- 개념: 조직 전체 통합 구조
- 내부: 물리 저장 구조

**함정**
- “개념 스키마는 사용자 관점이다” → ❌

### 4️⃣ 정규화·이상현상 기본 개념 연결
- 삽입·갱신·삭제 이상 방지 목적
- 반정규화는 성능 향상을 위한 구조적 타협

## 🧾 문제 풀이 전략 메모
1. “항상/반드시/오직” 절대 표현 등장 시 의심하기
2. 성능 향상만 강조하면 대체로 오답
3. 의미/구조/관계 중심으로 보기 분석하기
