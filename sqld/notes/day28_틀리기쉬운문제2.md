# **SQLD Day 28 – 틀리기 쉬운 문제 총정리 (2) / JOIN · 집합 · GROUPING (2025-11-09)** > 📘 SQLD 2과목 | SQL 기본 및 응용 > 🧑‍💻 학습자: Jaerok Kim (Raprok612) --- ## **🎯 학습 목표** - SQLD 실전에서 자주 등장하는 JOIN, 집합 연산, GROUPING 함수의 오답 유형 분석 - OUTER JOIN의 NULL 처리 및 WHERE 조건 위치별 결과 차이 이해 - UNION/UNION ALL/INTERSECT/MINUS의 중복 처리 규칙 복습 - GROUPING() 함수와 ROLLUP/CUBE 구조 완벽 숙지 --- ## **🧠 이론 정리** ### **1️⃣ JOIN 유형 정리** | JOIN 유형 | 설명 | 예시 | |------------|------|------| | **INNER JOIN** | 교집합 | SELECT * FROM A JOIN B ON A.ID = B.ID; | | **LEFT OUTER JOIN** | 왼쪽 기준, NULL 포함 | SELECT * FROM A LEFT JOIN B ON A.ID = B.ID; | | **RIGHT OUTER JOIN** | 오른쪽 기준 | SELECT * FROM A RIGHT JOIN B ON A.ID = B.ID; | | **FULL OUTER JOIN** | 양쪽 모두 포함 | SELECT * FROM A FULL OUTER JOIN B ON A.ID = B.ID; | > ⚠️ LEFT JOIN 후 WHERE절에서 오른쪽 테이블 조건을 주면 INNER JOIN처럼 작동한다.
sql
SELECT D.DEPTNO, D.DNAME, E.ENAME
FROM DEPT D
LEFT JOIN EMP E ON D.DEPTNO = E.DEPTNO
WHERE E.SAL > 2000;  -- ❌ INNER JOIN처럼 작동


💡 올바른 방식:

SELECT D.DEPTNO, D.DNAME, E.ENAME
FROM DEPT D
LEFT JOIN EMP E ON D.DEPTNO = E.DEPTNO AND E.SAL > 2000;  -- ✅ LEFT 유지

2️⃣ 집합 연산자 (Set Operators)
연산자	설명	중복 여부
UNION	합집합	중복 제거
UNION ALL	합집합	중복 포함
INTERSECT	교집합	중복 제거
MINUS (EXCEPT)	차집합	중복 제거

💡 집합 연산은 컬럼 개수, 데이터 타입, 순서가 모두 동일해야 한다.

SELECT DEPTNO FROM EMP
UNION
SELECT DEPTNO FROM DEPT;

3️⃣ GROUPING 함수와 ROLLUP/CUBE
함수	설명	예시
ROLLUP	계층형 합계	GROUP BY ROLLUP(DEPTNO, JOB)
CUBE	모든 조합 합계	GROUP BY CUBE(DEPTNO, JOB)
GROUPING()	집계 여부 표시	GROUPING(DEPTNO) → 1은 집계행

⚙️ GROUPING() 함수 결과는 BOOLEAN이 아닌 숫자형(1/0)
1: 집계행 / 0: 일반행

🧮 예제 및 실습

예제 1️⃣ – LEFT JOIN의 WHERE 조건 주의

SELECT D.DEPTNO, D.DNAME, E.ENAME
FROM DEPT D
LEFT JOIN EMP E ON D.DEPTNO = E.DEPTNO
WHERE E.ENAME IS NULL;


예제 2️⃣ – UNION vs UNION ALL

SELECT DEPTNO FROM EMP
UNION ALL
SELECT DEPTNO FROM DEPT;


예제 3️⃣ – GROUPING 함수 확인

SELECT DEPTNO, JOB, SUM(SAL),
       GROUPING(DEPTNO) AS G1, GROUPING(JOB) AS G2
FROM EMP
GROUP BY ROLLUP(DEPTNO, JOB);

💬 느낀점 / 메모

LEFT JOIN과 WHERE 조건 순서에 따라 결과가 완전히 달라진다는 점을 명확히 체득했다.

UNION과 UNION ALL의 차이는 단순 문법이 아닌 데이터 중복 처리 로직의 차이였다.

GROUPING 함수의 결과가 1/0이라는 점은 시험에서 함정 포인트로 자주 등장한다.
