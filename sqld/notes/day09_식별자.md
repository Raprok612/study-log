## **🧠 이론 정리**

### **1️⃣ 식별자의 개념**
식별자(Identifier)는 엔터티 내 인스턴스를 고유하게 구분하는 속성이다.

| 용어 | 설명 | 예시 |
|------|------|------|
| **본질식별자 (Natural Key)** | 실세계 업무에서 이미 존재하는 자연스러운 키 | 주민번호, 사업자등록번호, 이메일 |
| **인조식별자 (Surrogate Key)** | 업무와 무관하게 새로 생성된 내부 키 | USER_ID, ORDER_ID |

---

### **2️⃣ 본질식별자 vs 인조식별자**

| 구분 | 본질식별자 | 인조식별자 |
|------|-------------|-------------|
| **의미** | 업무 의미가 있는 값 | 의미 없는 시스템 번호 |
| **장점** | 자연스러움, 추가키 불필요 | 안정성, 변경 위험 적음 |
| **단점** | 변경 가능성, 개인정보 문제 | UNIQUE 제약 필요 |
| **예시** | 주민번호, 이메일 | 회원번호, 주문번호 |

---

### **3️⃣ 식별자 선정 기준**
- **불변성**: 시간이 지나도 변경되지 않을 것  
- **유일성**: 중복 발생 위험이 없을 것  
- **간결성**: 인덱스 길이가 적절할 것  
- **업무 독립성**: 업무 변경 시 PK가 영향받지 않을 것  

---

### **4️⃣ 적용 사례**

#### ✔ 본질식별자가 적절한 경우
- 자연적으로 고유한 값 존재  
- 변경 가능성 매우 낮음  

#### ✔ 인조식별자가 적절한 경우
- 변경 가능한 속성이 PK 후보일 때  
- 속성 조합으로 PK를 구성해야 할 때  
- 개인정보 노출 우려가 있을 때  

---

## **🧮 예제**

### ✔ 본질식별자 사용
```sql
CREATE TABLE MEMBER (
    SSN VARCHAR2(13) PRIMARY KEY,
    NAME VARCHAR2(50)
);
```

### ✔ 인조식별자 사용
```sql
CREATE TABLE MEMBER (
    MEMBER_ID NUMBER PRIMARY KEY,
    SSN VARCHAR2(13) UNIQUE,
    NAME VARCHAR2(50)
);
```

---

## **⚙️ 핵심 요약**
- 본질식별자 = 의미 있는 자연 키  
- 인조식별자 = 변경 위험 낮고 안정적  
- 실무에서는 인조식별자 + UNIQUE 본질속성 조합이 가장 일반적
