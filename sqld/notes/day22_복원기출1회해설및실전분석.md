## **🧠 이론 정리**

### **1️⃣ SELECT 실행 순서 복습**

| 순서 | 절(Clause) | 설명 |
|------|-------------|------|
| 1 | FROM | 데이터 소스 테이블 지정 |
| 2 | WHERE | 조건 필터링 |
| 3 | GROUP BY | 그룹화 기준 설정 |
| 4 | HAVING | 그룹 조건 |
| 5 | SELECT | 컬럼 및 표현식 선택 |
| 6 | ORDER BY | 정렬 |

> 💡 SQL은 ‘읽는 순서’와 ‘실행 순서’가 다르다. 실행 순서로 문제를 분석해야 함.

---

### **2️⃣ JOIN 응용 요약**

| 유형 | 설명 | 예시 |
|------|------|------|
| **INNER JOIN** | 두 테이블의 공통 데이터만 조회 | `A JOIN B ON A.ID = B.ID` |
| **LEFT OUTER JOIN** | 왼쪽 테이블 기준, 오른쪽 NULL 포함 | `A LEFT JOIN B ON A.ID = B.ID` |
| **RIGHT OUTER JOIN** | 오른쪽 기준 | `A RIGHT JOIN B ON A.ID = B.ID` |
| **SELF JOIN** | 자기 자신과 JOIN | `EMP E1 JOIN EMP E2 ON E1.MGR = E2.EMPNO` |

> ⚙️ 복원 기출에서는 LEFT JOIN + NULL 처리 조합 문제 다수 출제됨.

---

## **🧮 예제 및 실습**

**예제 1️⃣ – LEFT JOIN + GROUP BY**
```sql
SELECT D.DEPT_NAME, COUNT(E.EMP_ID) AS CNT
FROM DEPT D
LEFT JOIN EMP E ON D.DEPT_ID = E.DEPT_ID
GROUP BY D.DEPT_NAME
ORDER BY CNT DESC;
```

**예제 2️⃣ – 서브쿼리 + JOIN 결합**
```sql
SELECT E.ENAME, E.SAL
FROM EMP E
WHERE E.SAL > (
    SELECT AVG(SAL) FROM EMP WHERE DEPTNO = E.DEPTNO
);
```

**예제 3️⃣ – GROUP BY + HAVING**
```sql
SELECT DEPTNO, AVG(SAL)
FROM EMP
GROUP BY DEPTNO
HAVING AVG(SAL) >= 3000;
```

## **💬 느낀점 / 메모**

- SELECT 실행 순서에 따라 문제를 해석하니 정답 도출이 훨씬 빨라졌다.  
- LEFT JOIN과 NULL 처리의 조합이 실제 시험에서도 자주 등장할 가능성이 높다.  
- GROUP BY와 HAVING의 순서를 완벽히 체화해야 함을 느꼈다.
